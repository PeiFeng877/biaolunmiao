# Testing Execution Policy

[PROTOCOL]: 变更时更新此头部，然后检查 agents.md

**版本**: v1.0  
**日期**: 2026-02-17  
**定位**: BianLunMiao iOS 项目的测试执行分层规范（快检 + 全检）。

## 1. 目标定义
1. 质量底线: 关键流程不可被小改动破坏。
2. 反馈速度: 开发提交后 5-10 分钟内获得有效反馈。
3. 风险匹配: 小改动跑小检查，大改动跑大回归。

## 2. 执行分层
### 2.1 快检（PR 主门禁）
- Workflow: `.github/workflows/design-governance.yml`
- 触发: `pull_request`、非 `main/master` 的 `push`
- 必跑内容:
  - 治理脚本检查（按钮/反馈 API 一致性）
  - SwiftFormat（`--lint --lenient`）
  - SwiftLint（`.swiftlint.yml`）
  - 单元测试（`-only-testing:BianLunMiaoTests`）
- 风险追加:
  - 当改动触达 UI 风险路径（`Views/DesignSystem/ViewModels/Data/Models`）时，追加 UI 冒烟:
    - `BianLunMiaoUITests/testExample`
    - `BianLunMiaoUITests/testScheduleMonthToDayDetailAndSourceManagement`

### 2.2 全检（主干健康 / 发布保障）
- Workflow: `.github/workflows/full-regression.yml`
- 触发:
  - `main/master` 分支 push
  - 每日定时（UTC `19:00`）
  - 手动触发（`workflow_dispatch`）
- 执行内容:
  - 治理脚本
  - SwiftFormat/SwiftLint
  - 全量 `xcodebuild test`（不做 `only-testing` 限制）

## 3. 规范要求
1. 新增测试默认进入 `BianLunMiaoTests`（业务逻辑、状态、边界）；UI 自动化仅放 `BianLunMiaoUITests`。
2. 快检失败不可合并；全检失败不可发布。
3. 任何放宽规则（例如 lint 规则、UI 冒烟范围）必须在本文件追加变更记录。
4. 治理脚本始终属于 Hard Gate，不能降级为告警。

## 4. 后续优化路线
1. 将 SwiftFormat/SwiftLint 从“告警可见”逐步升级到“严格阻断”。
2. 增加 UI 冒烟截图自动归档（稳定证据链）。
3. 依据模块风险动态扩展 UI 冒烟白名单。
