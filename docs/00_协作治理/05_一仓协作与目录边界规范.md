# 一仓协作与目录边界规范

[PROTOCOL]: 变更时更新此头部，然后检查 agents.md

**版本**: v2.1
**日期**: 2026-03-14

## 1. 目标
1. 建立单仓模式下文档、iOS、后端目录的稳定协作边界。
2. 保证“一个仓库统一提交、按目录治理职责”的一致性。

## 2. 仓库边界
1. 主仓（`./`）: 统一维护跨端 SSOT 文档、iOS 客户端代码、后端服务代码、治理脚本与历史归档。
2. iOS 目录（`./bianlunmiao-ios/`）: 维护 iOS 客户端实现与 iOS 端专属文档。
3. 后端目录（`./辩论喵-后端/`）: 维护后端 API、模型、迁移与端内文档。
4. `docs/` 仍为跨端 SSOT；`bianlunmiao-ios/docs/` 为 iOS 端专属文档区。
5. 仓库内不再允许存在嵌套 `.git` 或额外独立 Git 根目录。

## 3. 变更流程
1. 端内需求: 在主仓内修改对应目录，并执行该目录约定的最小验证。
2. 跨端契约需求: 先更新 `docs/03_接口与数据契约/`，再推进 iOS 或后端实现。
3. 治理规则需求: 在主仓直接更新根 `agents.md`、`docs/00_协作治理/` 与相关 L2 文档。
4. 发布需求: iOS 与后端仍按各自流程打包验证，但提交历史统一保留在主仓。

## 3.1 提交边界矩阵（强制）
1. 同一功能涉及 `docs/**`、`bianlunmiao-ios/**`、`辩论喵-后端/**` 时，允许同仓同提交。
2. 不同主题的改动必须拆分提交，哪怕都在同一仓。
3. 纯治理改动应避免混入业务实现代码。
4. 纯业务改动若影响契约、测试或发布语义，必须同步回写文档。

## 4. 同构与回写
1. 任一目录新增/删除/重命名结构时，必须同步更新该目录 `agents.md`。
2. 影响跨端协作语义时，必须同步回写根 `agents.md` 与 `docs/`。
3. 文档冲突时，以根仓 SSOT 目录最新版本为准，再触发端内目录修正。

## 5. 验收标准
1. `git ls-files` 应同时包含 `docs/`、`bianlunmiao-ios/`、`辩论喵-后端/`。
2. `find . -maxdepth 2 -name .git -type d` 仅应返回根仓 `.git`。
3. 根仓文档满足 `[PROTOCOL]`、版本、日期、变更日志。
4. 提交前已执行：`pwd`、`git rev-parse --show-toplevel`、`git status --short`。
