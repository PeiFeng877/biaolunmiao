# 多仓协作与边界规范

[PROTOCOL]: 变更时更新此头部，然后检查 agents.md

**版本**: v1.2
**日期**: 2026-03-14

## 1. 目标
1. 建立根仓与 iOS/后端子仓的稳定协作边界。
2. 保证“端内实现独立迭代、跨端契约集中治理”的一致性。

## 2. 仓库边界
1. 根仓（`./`）: 仅维护跨端 SSOT 文档、治理脚本、协作规范。
2. iOS 子仓（`./bianlunmiao-ios/`）: 仅维护 iOS 客户端代码与 iOS 端文档。
3. 后端子仓（`./辩论喵-后端/`）: 仅维护后端服务代码与后端端内文档。
4. 根仓不使用 submodule，仅通过目录共址协作。

## 3. 变更流程
1. 端内需求: 在对应子仓提交并验证门禁。
2. 跨端契约需求: 先更新根仓 `docs/03_接口与数据契约/`，再推进子仓实现。
3. 治理规则需求: 在根仓提交治理文档，再同步子仓 `agents.md`。
4. 发布需求: 各子仓独立打包与验证，根仓只保留发布流程规范与记录。

## 3.1 提交路由矩阵（强制）
1. iOS 改动（`bianlunmiao-ios/**`）-> iOS 子仓提交。
2. 后端改动（`辩论喵-后端/**`）-> 后端子仓提交。
3. 协作治理与跨端文档改动（根 `agents.md`、`docs/**`、`scripts/**`）-> 根仓提交。
4. 混合改动 -> 拆分到各自仓库提交，禁止跨仓混合提交。

## 4. 同构与回写
1. 任一子仓新增/删除/重命名结构时，必须同步更新该子仓 `agents.md`。
2. 影响跨端协作语义时，必须同步回写根仓 `agents.md` 与 `docs/`。
3. 根仓与子仓文档冲突时，以根仓 SSOT 目录最新版本为准，再触发子仓修正。

## 5. 验收标准
1. 根仓 `git ls-files` 不包含三端源码目录文件。
2. iOS 与后端子目录各自存在独立 `.git`，并可独立执行 `git status`。
3. 根仓文档满足 `[PROTOCOL]`、版本、日期、变更日志。
4. 任一仓提交前均已执行：`pwd`、`git rev-parse --show-toplevel`、`git status --short`。
