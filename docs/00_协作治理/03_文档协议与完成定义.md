## 1. 文档协议
1. 所有规范文档必须包含 `[PROTOCOL]` 头。
2. 必须包含版本、日期、变更日志。
3. 文档结构变化时，同步更新所在目录 `agents.md`。
4. 文档与代码冲突时，先修复冲突再继续开发。

## 2. 结构约束（复用 iOS 既有规则）
1. 建议单文件不超过 800 行。
2. 建议单层目录文件数不超过 8 个。
3. 超限时按业务子域拆分。

## 3. 完成定义（DoD）
1. 需求边界明确。
2. 契约文档已更新。
3. 代码实现可运行。
4. 最小测试通过。
5. 回滚方案可执行。
6. 变更日志已回写。
