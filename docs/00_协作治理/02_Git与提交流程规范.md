# Git与提交流程规范

[PROTOCOL]: 变更时更新此头部，然后检查 agents.md

**版本**: v1.2
**日期**: 2026-03-05

## 1. 分支规范
1. 分支命名统一使用 `codex/<topic>`。
2. 默认短分支开发，尽快合并。
3. 禁止在主干长期堆积未审查变更。

## 2. 提交规范
1. 提交格式：`feat|fix|docs|refactor: <scope> <summary>`。
2. 一次提交聚焦一个主题，避免混杂无关改动。
3. 涉及契约变更时，提交必须包含文档更新。

## 3. PR 规范
1. PR 描述必须包含：影响模块、接口变化、测试结果、回滚方案。
2. 跨端 PR 必须附带契约文档引用。
3. 门禁失败不得合并。

## 4. 发布前检查
1. iOS/后端最小测试通过。
2. 文档协议检查通过（版本、日期、变更日志）。
3. 关键路径支持回滚。

## 5. 多仓协作约束
1. 根仓仅跟踪 `docs/`、`scripts/` 与治理文件，不跟踪 `bianlunmiao-ios/`、`bianlunmiao-flutter/`、`辩论喵-后端/` 源码。
2. 端内改动必须在对应子仓提交；根仓提交只承载跨仓规则、契约与治理更新。
3. 涉及跨端契约变化时，先在根仓 `docs/03_接口与数据契约/` 更新，再在各子仓实现。
4. 同一需求跨多个仓库时，提交说明必须显式给出“根仓提交 + 子仓提交”的对应关系。

## 6. 提交归属判定（强制）
1. `bianlunmiao-ios/**` 改动：只允许在 iOS 子仓提交。
2. `辩论喵-后端/**` 改动：只允许在后端子仓提交。
3. `bianlunmiao-flutter/**` 改动：只允许在 Flutter 子仓提交。
4. `docs/**`、根 `agents.md`、根 `scripts/**` 改动：只允许在根仓提交。
5. 发现“当前仓库职责外”文件出现在 `git status`，必须停止提交并切换到正确仓库处理。

## 7. 提交前检查清单（强制）
1. `pwd`：确认当前目录。
2. `git rev-parse --show-toplevel`：确认当前 Git 根。
3. `git status --short`：确认提交集仅包含当前仓库职责文件。
4. 若同时改动多个仓库：分别进入各仓库提交，不得用单仓一次性提交全部改动。
