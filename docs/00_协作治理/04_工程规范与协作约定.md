# 工程规范与协作约定

[PROTOCOL]: 变更时更新此头部，然后检查 agents.md

**版本**: v1.0
**日期**: 2026-02-17


## 1. 分层规范

- `api/`：仅处理 HTTP 协议、鉴权、参数校验、响应装配。
- `models/`：数据库实体定义。
- `schemas/`：请求/响应 DTO。
- `services/`：业务规则工具与跨域逻辑。
- `db/`：数据库连接、会话、迁移入口。

## 2. API 规范

1. 统一前缀：`/api/v1`。
2. 写接口返回最新对象快照。
3. 列表统一：`{ items, nextCursor }`。
4. 统一错误体：`{ code, message, requestId, details? }`。
5. 时间字段统一 UTC ISO8601。

## 3. 错误码规范

- 鉴权：`UNAUTHORIZED`、`INVALID_TOKEN`
- 权限：`FORBIDDEN`、`TEAM_ROLE_FORBIDDEN`
- 业务冲突：`CONFLICT`、`DUPLICATE_PENDING_REQUEST`
- 数据校验：`VALIDATION_ERROR`
- 不存在：`NOT_FOUND`

## 4. 数据规范

1. 主键统一 UUID 字符串（36 位）。
2. 对外展示 ID 统一 `public_id`（用户/队伍）。
3. 使用软状态字段（`status`）描述生命周期。

## 5. Git 与发布规范

1. 分支命名：`codex/<feature-or-fix>`。
2. 提交信息：`feat|fix|docs|refactor: <scope> <summary>`。
3. 禁止直接在生产改配置，必须经预发验证。

## 6. 测试规范

- 新增核心业务规则必须带最少 1 个接口级测试。
- 每次发布前至少验证：认证、队伍申请、场次排阵、赛果录入、消息 ack。
