# 错误码与响应规范

[PROTOCOL]: 变更时更新此头部，然后检查 agents.md

**版本**: v1.4
**日期**: 2026-03-19

## 1. 响应结构
1. 错误响应统一为：`{ code, message, requestId, details? }`。
2. 列表响应统一为：`{ items, nextCursor }`。
3. 写接口默认返回“最新对象快照”。

## 2. 错误码分层
1. 鉴权类：`UNAUTHORIZED`、`INVALID_TOKEN`、`ACCOUNT_DELETED`、`APPLE_TOKEN_INVALID`。
   `ACCOUNT_DELETED` 仅用于访问已删除账号的受保护接口等场景，不再用于 `/auth/apple` 重新注册。
   `APPLE_TOKEN_INVALID` 统一覆盖 Apple token 结构错误、签名错误、`sub` 非法或超出持久化上限等场景，不允许下沉为 `500`。
2. 权限类：`FORBIDDEN`、`TEAM_ROLE_FORBIDDEN`。
3. 冲突类：`CONFLICT`、`DUPLICATE_PENDING_REQUEST`。
4. 校验类：`VALIDATION_ERROR`。
5. 资源类：`NOT_FOUND`。

## 3. 时间与标识
1. 时间字段统一使用 UTC ISO8601。
2. 主键统一 UUID（36 位字符串）。
3. 对外展示标识统一 `public_id`。

## 4. 变更规则
1. 新增错误码必须更新本文件与 API 契约文档。
2. 不允许复用旧错误码表达新语义。
3. 删除错误码前必须确认客户端已完成兼容。
