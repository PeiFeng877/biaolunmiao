# 错误码与响应规范

[PROTOCOL]: 变更时更新此头部，然后检查 agents.md

**版本**: v1.8
**日期**: 2026-03-22

## 1. 响应结构
1. 错误响应统一为：`{ code, message, requestId, details? }`。
2. 列表响应统一为：`{ items, nextCursor }`。
3. 写接口默认返回“最新对象快照”。
4. 迁移到 `FC + FastAPI` 后，错误体字段名和时间格式不得变化。

## 2. 错误码分层
1. 鉴权类：`UNAUTHORIZED`、`INVALID_TOKEN`、`ACCOUNT_DELETED`、`APPLE_TOKEN_INVALID`、`DEBUG_TOKEN_DISABLED`、`PHONE_INVALID`、`PHONE_CODE_INVALID`、`PHONE_CODE_EXPIRED`、`PHONE_CODE_TOO_FREQUENT`、`PHONE_AUTH_NOT_AVAILABLE`。
   `ACCOUNT_DELETED` 仅用于访问已删除账号的受保护接口等场景，不再用于 `/auth/apple` 重新注册。
   `APPLE_TOKEN_INVALID` 统一覆盖 Apple token 结构错误、签名错误、`sub` 非法或超出持久化上限等场景，不允许下沉为 `500`。
   `DEBUG_TOKEN_DISABLED` 用于正式环境或关闭调试开关时访问 `POST /auth/debug-token`。
   `PHONE_INVALID` 用于手机号格式非法、非中国大陆号码或标准化失败。
   `PHONE_CODE_INVALID` 用于验证码不正确、请求不存在或 provider 校验失败。
   `PHONE_CODE_EXPIRED` 用于验证码过期。
   `PHONE_CODE_TOO_FREQUENT` 用于短时间内重复发送验证码。
   `PHONE_AUTH_NOT_AVAILABLE` 用于短信认证 provider 不可用、配置缺失或正式环境误启 mock。
2. 权限类：`FORBIDDEN`、`TEAM_ROLE_FORBIDDEN`。
3. 冲突类：`CONFLICT`、`DUPLICATE_PENDING_REQUEST`。
4. 校验类：`VALIDATION_ERROR`。
5. 资源类：`NOT_FOUND`。

## 3. 时间与标识
1. 时间字段统一使用 UTC ISO8601。
2. 主键统一 UUID（36 位字符串）。
3. 对外展示标识统一 `public_id`。

## 4. 变更规则
1. 新增错误码必须更新本文件与 API 契约文档。
2. 不允许复用旧错误码表达新语义。
3. 删除错误码前必须确认客户端已完成兼容。
