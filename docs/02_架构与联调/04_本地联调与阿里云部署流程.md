**适用阶段**: iOS 为主交付端，Flutter 迁移推进中，后端持续联调与发布。

---

## 1. 推荐技术方案（可直接落地）

为兼顾开发效率、国内部署与后续 Flutter 多端复用，建议：
1. API：FastAPI（或 NestJS，二选一，本文按 FastAPI 说明）。
2. DB：PostgreSQL（主数据）。
3. 缓存：Redis/Tair（会话、限流、热点数据）。
4. 对象存储：OSS（头像/封面图）。
5. 运行环境：容器化（Docker）。

---

## 2. 本地开发测试流程（对应你的 3(a)）

结论：
1. 建议“独立后端服务 + 独立数据库 + 独立缓存”本地启动。
2. iOS 不再直连本地 MockData，改为走 HTTP API。

## 2.1 本地最小拓扑
1. `api`（FastAPI）
2. `postgres`（PostgreSQL）
3. `redis`（Redis）
4. `minio`（本地替代 OSS，可选）

## 2.2 开发流程（推荐）
1. 启动容器：`docker compose up -d`
2. 执行迁移：`alembic upgrade head`
3. 初始化种子数据（开发账号、队伍、赛事）。
4. 启动 API 热更新服务（本机 8000 端口）。
5. iOS Simulator 指向 `http://127.0.0.1:8000`。
6. iOS 真机调试指向电脑局域网 IP（如 `http://192.168.x.x:8000`）。
7. 每次提交前跑：
- API 单测
- 集成测试（队伍、申请、赛事、赛程核心链路）
- OpenAPI 契约校验

## 2.3 环境变量分层
1. `local`: 本地 compose 默认配置。
2. `staging`: 云端预发配置（阿里云资源）。
3. `prod`: 生产配置（强制只读密钥、审计日志开启）。

---

## 3. 阿里云部署流程（对应你的 3(b)）

你面向国内用户，建议采用“分阶段上云”：
1. MVP 阶段：ECS + Docker Compose（最快上线）。
2. 稳定阶段：ACK/SAE + ACR + 托管数据库（可扩展）。

## 3.1 上线前准备
1. 确认地域（建议靠近主要用户，如华东/华北）。
2. 域名购买与备案（国内访问必须考虑备案周期）。
3. RAM 账号与最小权限策略（禁止长期使用主账号 AK）。

## 3.2 基础资源清单
1. VPC + 子网 + 安全组。
2. ECS（运行 API，MVP 可单机）。
3. RDS PostgreSQL（生产建议托管）。
4. Tair/Redis（缓存与会话）。
5. OSS（媒体文件）。
6. ACR（容器镜像仓库）。
7. ALB/Nginx + HTTPS（统一入口）。
8. 日志与监控（SLS + CloudMonitor）。

## 3.3 MVP 阶段（ECS 方案）
1. 在 ACR 建仓，推送 API 镜像。
2. ECS 安装 Docker/Compose。
3. 在 ECS 拉起 `api` 容器，数据库与缓存用 RDS/Tair 托管实例。
4. Nginx 反向代理到 API，证书接入 HTTPS。
5. 开放白名单端口，仅暴露 443。
6. 配置定时备份与告警（CPU、内存、连接数、错误率）。

## 3.4 进阶阶段（ACK/SAE）
1. 使用 ACR + ACK/SAE 托管应用实例。
2. HPA 自动扩缩容（按 CPU/QPS）。
3. 滚动发布 + 灰度发布。
4. 数据层保持 RDS/Tair/OSS 托管。

## 3.5 建议的环境分层
1. `dev`：开发环境（低配、低成本）。
2. `staging`：预发布环境（与 prod 同拓扑，小一档规格）。
3. `prod`：生产环境（高可用、强监控、备份与恢复演练）。

---

## 4. 与 iOS 联调的云端流程

1. 后端先发布到 `staging` 域名（如 `api-staging.xxx.com`）。
2. iOS 新增环境切换（Debug 指向 staging，Release 指向 prod）。
3. 联调时固定 API 版本前缀（`/api/v1`），避免前后端错位。
4. 每次后端变更同步 OpenAPI 文档并通知 iOS/Flutter。

---

## 5. 风险与规避

1. 风险：备案周期导致无法按时上线。
- 规避：尽早备案，先用测试域名联调。
2. 风险：全部服务上容器编排，早期运维负担偏高。
- 规避：MVP 先 ECS 单机 + 托管数据库，后续再 ACK 化。
3. 风险：客户端直接依赖后端临时字段。
- 规避：严格契约治理，接口字段加版本管理。
