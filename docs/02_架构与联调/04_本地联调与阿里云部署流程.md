# 后端实现方案：本地联调、FC 部署与新 RDS Serverless 说明

**版本**: v2.0
**日期**: 2026-03-22
**适用阶段**: FC 单正式环境、本地联调与新购 RDS Serverless 采购准备。

## 1. 文档定位
1. 本文定义现行的本地联调、FC 部署与云资源采购流程。
2. 现行目标架构以 `05_FC单正式环境与本地开发方案.md` 为准。
3. 本文只保留三类信息：
   - 本地 FastAPI 回放口径
   - FC 默认域名正式承接口径
   - 新购 RDS PostgreSQL Serverless 的采购与切换口径

## 2. 本地联调口径
1. 当前本地回放使用 Python FastAPI 后端：
   - 目录：`辩论喵-后端/`
   - 基址：`http://127.0.0.1:8788`
2. 本地入口只用于开发回放与最小烟测。
3. iOS 本地联调只验证当前主流程，不再分出云端测试服。

## 3. 现行资源对照
1. 现行资源：
   - 计算：`FC Web 函数 + FastAPI`
   - 数据：新购杭州 `RDS PostgreSQL Serverless`
   - 存储：`OSS`
   - 入口：FC 默认域名
2. 旧 `SAE`、`CLB`、`ECS/Caddy` 与旧 `EMAS` 仅保留为历史记录，不再作为回退路径。
3. 旧 `RDS` 继续运行到新库可用为止，但不迁历史数据。

## 4. 部署建议
1. 先冻结 `GET /healthz` 与 `POST /api` 契约、动作命名与错误码，再推进正式部署。
2. 先把本地 FastAPI 跑通，再部署到 FC 默认域名做正式 smoke。
3. `api.bianlunmiao.top` 本轮不切，等你后续通知再接正式自定义域名。

## 5. 新 RDS 采购记录
### 5.1 采购目标
1. 地域：`cn-hangzhou`
2. 引擎：`PostgreSQL`
3. 计费方式：`Serverless`
4. 自动启停：`AutoPause` 开启
5. 规格：购买页可选的最小 Serverless 配置

### 5.2 切换原则
1. 新库只承接新 schema 和新种子，不迁旧数据。
2. 新库可用后再释放旧 RDS。
3. `TestFlight` 与正式后端共用新库，但必须使用专用测试账号与命名规范。
