# 客户端现状与联调基线

[PROTOCOL]: 变更时更新此头部，然后检查 agents.md

**版本**: v1.5
**日期**: 2026-03-22
**适用范围**: `bianlunmiao-ios/`、`bianlunmiao-admin/`、`辩论喵-后端/`

## 1. iOS 现状
1. 产品主流程（队伍/赛事/日程/消息/我的）已具备可运行闭环。
2. 设计系统与治理脚本已建立，按钮/反馈组件具备可审计清单。
3. 发布侧已完成正式包预检与重新提审记录，当前以 iOS 为主发布端。

## 2. 后端联调现状
1. 后端当前目标是 `FC Web 函数 + FastAPI`，不再保留云端测试服。
2. 当前重点是把本地 FastAPI 跑通，再部署到 FC 默认域名，客户端通过数据层改造吸收协议迁移。
3. 联调验收以接口契约、错误语义、连通性检查与自动化测试结果对齐为核心。

## 3. 联调基线
1. 客户端默认以契约版本驱动，不以端内临时实现定义服务端语义。
2. Release 语义统一走正式域名根地址；Debug 默认按运行载体切换为“模拟器走本机后端、真机走 FC 默认域名”，具体环境变量与启动参数解析统一由 `RuntimeOverrides.swift` 承接，仍允许通过显式覆盖。
3. 任何登录、权限、错误码语义变化都必须先更新根仓契约文档。

## 4. 验收门禁
1. iOS：治理检查 + lint/test + 发布预检。
2. 后端：本机健康检查、关键接口烟测、FC 默认域名 smoke、数据库初始化验证。
3. 联调：关键业务路径在本机与正式域名环境可重复验证；Web 管理后台纳入本轮联合验收。
