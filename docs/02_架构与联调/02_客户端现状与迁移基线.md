# 客户端现状与联调基线

[PROTOCOL]: 变更时更新此头部，然后检查 agents.md

**版本**: v1.1
**日期**: 2026-03-14
**适用范围**: `bianlunmiao-ios/`、`辩论喵-后端/`

## 1. iOS 现状
1. 产品主流程（队伍/赛事/日程/消息/我的）已具备可运行闭环。
2. 设计系统与治理脚本已建立，按钮/反馈组件具备可审计清单。
3. 发布侧已完成正式包预检与重新提审记录，当前以 iOS 为主发布端。

## 2. 后端联调现状
1. 后端已承接 iOS 主流程所需的 MVP 接口域，并具备本机联调与云端预发能力。
2. 当前重点是稳定 iOS 与后端的真实链路，而不是扩展新的客户端实现。
3. 联调验收以接口契约、错误语义与自动化测试结果对齐为核心。

## 3. 联调基线
1. 客户端默认以契约版本驱动，不以端内临时实现定义服务端语义。
2. Release 语义统一走正式域名；联调通过显式环境变量切换。
3. 任何登录、权限、错误码语义变化都必须先更新根仓契约文档。

## 4. 验收门禁
1. iOS：治理检查 + lint/test + 发布预检。
2. 后端：`make lint`、`make test`。
3. 联调：关键业务路径在本机后端与 iOS 模拟器环境可重复验证。
