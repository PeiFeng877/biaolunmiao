# 客户端现状与迁移基线

[PROTOCOL]: 变更时更新此头部，然后检查 agents.md

**版本**: v1.0
**日期**: 2026-03-05
**适用范围**: `bianlunmiao-ios/`、`bianlunmiao-flutter/`

## 1. iOS 现状
1. 产品主流程（队伍/赛事/日程/消息/我的）已具备可运行闭环。
2. 设计系统与治理脚本已建立，按钮/反馈组件具备可审计清单。
3. 发布侧已完成正式包预检与重新提审记录，当前以 iOS 为主发布端。

## 2. Flutter 迁移现状
1. Flutter 仓库已建立并接管 Android 方向后续实现。
2. 当前策略是“语义同构迁移”，不在迁移阶段引入新的产品规则。
3. 迁移验收以跨端同构清单与接口契约对齐为核心。

## 3. 联调基线
1. 客户端默认以契约版本驱动，不以端内临时实现定义服务端语义。
2. Release 语义统一走正式域名；联调通过显式环境变量切换。
3. 任何登录、权限、错误码语义变化都必须先更新根仓契约文档。

## 4. 验收门禁
1. iOS：治理检查 + lint/test + 发布预检。
2. Flutter：`flutter analyze`、`flutter test`、`flutter build apk --debug`。
3. 跨端：同构验收脚本通过且关键业务路径可互通。
