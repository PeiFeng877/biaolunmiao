# 前后端服务边界与对接方案

[PROTOCOL]: 变更时更新此头部，然后检查 agents.md

**版本**: v1.0
**日期**: 2026-03-04
**适用范围**: `bianlunmiao-ios/`、`辩论喵-后端/` 跨端联调与正式环境对接

## 1. 文档目标
1. 统一 iOS 客户端与后端服务的职责边界，避免“前后端各自兜底”。
2. 规定接口契约、环境切换与联调门禁的唯一归口。
3. 为正式发布、回归测试与中国大陆上架准备提供可追踪边界。

## 2. 边界原则
1. 接口字段、错误码、时间格式以 `docs/03_接口与数据契约/` 为 SSOT。
2. iOS 只负责展示、交互编排、本地轻状态与系统权限，不复制服务端业务判定。
3. 后端负责身份校验、持久化、权限判定、业务规则、媒体签名与可审计日志。
4. 任何跨端字段新增、语义变化或鉴权规则变化，必须先改文档再改实现。

## 3. iOS 端职责
1. 应用入口、Apple 登录门禁、资料完善分流与主导航编排。
2. 本地 UI 状态、加载态/错误态呈现与最小离线兜底。
3. 调用系统能力，如日历权限、链接跳转、Apple 登录授权。
4. 根据契约组装请求，不擅自生成服务端未定义字段。
5. 使用 Release 默认基址 `https://api.bianlunmiao.top/api/v1`，测试场景通过环境变量显式切换。

## 4. 后端职责
1. 校验 Apple identity token 并签发业务 access token。
2. 承载用户、队伍、赛事、日程、消息与媒体上传相关接口。
3. 统一输出错误码、时间格式、分页与鉴权失败语义。
4. 维护 Staging/Prod 环境配置、数据库迁移、对象存储签名与日志观测。
5. 在正式环境关闭 debug-token 等测试专用入口。

## 5. 对接基线
1. 鉴权:
   - iOS 发起 Apple 授权。
   - 后端完成 token 校验与新老用户识别。
   - 客户端依据 `isNewUser` 等契约字段做资料完善分流。
2. 业务数据:
   - 列表、详情、编辑提交均由后端作为事实源。
   - 客户端不缓存会改变业务结果的派生规则。
3. 媒体上传:
   - 客户端只拿预签名参数并上传对象。
   - 存储路径、签名、公开访问策略由后端控制。

## 6. 环境与联调约束
1. 本机联调基址：`http://127.0.0.1:8000/api/v1`
2. Staging 临时入口与联调流程以 `docs/04_测试与发布/02_记录/06_云端部署实施追踪_MVP.md` 为准。
3. Release/TestFlight/App Store 一律指向正式 HTTPS 域名，不允许带测试开关出包。
4. 自动化联调可跳过 Apple 登录，但真机正式验收必须执行一次真实 Apple 登录。

## 7. 变更触发器
1. 新增/删除 API、响应字段、错误码。
2. 登录方式、会话恢复、资料完善流程变化。
3. 域名、环境拓扑、对象存储或鉴权链路变化。
4. 中国大陆备案、隐私、提审要求影响客户端或服务端行为。
