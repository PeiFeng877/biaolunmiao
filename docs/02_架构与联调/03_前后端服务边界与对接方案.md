# 前后端服务边界与对接方案

[PROTOCOL]: 变更时更新此头部，然后检查 agents.md

**版本**: v1.3
**日期**: 2026-03-22
**适用范围**: `bianlunmiao-ios/`、`辩论喵-后端/` 跨端联调与正式环境对接

## 1. 文档目标
1. 统一客户端与后端服务的职责边界，避免“前后端各自兜底”。
2. 规定接口契约、环境切换与联调门禁的唯一归口。
3. 为 FC 单正式环境、本地开发与正式切流提供可追踪边界。

## 2. 边界原则
1. 接口字段、错误码、时间格式以 `docs/03_接口与数据契约/` 为 SSOT。
2. iOS 只负责展示、交互编排、本地轻状态与系统权限，不复制服务端业务判定。
3. Web 管理后台纳入本轮正式发布范围，线上对接边界统一通过 `Vercel prod + /api/proxy` 定义。
4. 后端负责身份校验、持久化、权限判定、业务规则、媒体签名与可审计日志。
5. 任何跨端字段新增、语义变化或鉴权规则变化，必须先改文档再改实现。

## 3. 客户端职责
1. 应用入口、Apple 登录门禁、资料完善分流与主导航编排。
2. 本地 UI 状态、加载态/错误态呈现与最小离线兜底。
3. 调用系统能力，如日历权限、链接跳转、Apple 登录授权。
4. 根据契约组装请求，不擅自生成服务端未定义字段。
5. 使用 Release 默认根地址对应的 FC 默认域名，健康检查走 `/healthz`，业务请求统一走 `POST /api`。

## 4. 后端职责
1. 目标运行时统一为 `FC Web 函数 + FastAPI`，不再保留云端测试服。
2. 承载用户、队伍、赛事、日程、消息、媒体上传相关动作。
3. 统一输出错误码、时间格式、分页与鉴权失败语义。
4. 正式环境使用 FC 默认域名承接流量，新购杭州 `RDS PostgreSQL Serverless` 与现有 `OSS` 作为正式资源。
5. 在正式环境关闭 debug-token 等测试专用入口。

## 5. 对接基线
1. 鉴权:
   - iOS 发起 Apple 授权。
   - 后端完成 token 校验与新老用户识别。
   - 客户端依据 `isNewUser` 等契约字段做资料完善分流。
2. 业务数据:
   - 列表、详情、编辑提交均由后端作为事实源。
   - 客户端不缓存会改变业务结果的派生规则。
3. 媒体上传:
   - 客户端只拿预签名参数并上传对象。
   - 存储路径、签名、公开访问策略由后端控制。

## 6. 环境与联调约束
1. 本机联调基址：`http://127.0.0.1:8788`，健康检查为 `/healthz`，业务入口为 `POST /api`。
2. 正式联调基址：FC 默认域名，iOS Debug 仅通过显式覆盖切换，不再依赖云端测试服。
3. Release/TestFlight/App Store 与 Web prod 一律指向正式入口，不允许带测试开关出包。
4. 自动化联调可跳过 Apple 登录，但真机正式验收必须执行一次真实 Apple 登录。
5. TestFlight 共享正式后端与数据库，但必须使用专用测试账号与命名规范。

## 7. 变更触发器
1. 新增/删除 API、响应字段、错误码。
2. 登录方式、会话恢复、资料完善流程变化。
3. 域名、环境拓扑、对象存储或鉴权链路变化。
4. FC 发布流程、RDS Serverless 规格或 AutoPause 策略变化。
5. 中国大陆备案、隐私、提审要求影响客户端或服务端行为。
