## 5. 风险与阻塞
1. Web 登录态、prod 绑定与正式部署均已恢复，旧 `api.bianlunmiao.top` 口径已从现役正式部署中剔除。
2. 后端正式短信配置已补齐并完成 deploy；本地直接调用阿里云号码认证 SDK 时，`SendSmsVerifyCode` 已能走到供应商侧并返回业务级频控错误 `biz.FREQUENCY / check frequency failed`，说明 AK、签名、模板和参数格式已基本可用。
3. 当前系统级发布阻塞已消除；手机号验证码链路恢复到供应商业务级返回，使用真实测试号码时仍需注意阿里云频控与测试节奏。
4. “先输错一次再输对一次”这条路径已改为服务端本地摘要核验，并补充了服务层回归测试；注意本轮热修前已下发的旧验证码不会继续生效，必须重新获取一条新验证码后再复验。
5. 现网 `FC` 曾出现冷启动时连库超时，已通过改用 RDS 内网域名、增大 `connect_timeout` 和重启 RDS 做过缓解；当前 `healthz` 与 RPC 已稳定通过。
6. 当前 `MEDIA_BACKEND=local` 且 `OSS_*` 为空，头像上传链路仍未达到“正式 OSS 落盘”目标状态；本轮未将该项作为阻塞 TestFlight 的条件。
7. `TestFlight` 与正式后端共用同一套服务与数据库，必须使用专用测试账号与测试命名规范；本轮 `1.0.1 (103)` 已上传并分发到内部测试组。
8. 本机对 `api.appstoreconnect.apple.com` 仍存在间歇性 DNS 解析异常；本轮通过临时 hosts 固定解析完成上传与分发，当前已清理该临时映射。
9. 正式入口以 `FC` 默认域名为准，禁止在发布说明、App 元数据或管理后台配置中混用 `api.bianlunmiao.top` 历史口径。
