# 阿里云资源申请指引（MVP 1.0，小白版）

[PROTOCOL]: 变更时更新此头部，然后检查 agents.md

**版本**: v2.0
**日期**: 2026-03-22

> 目标人群：非技术背景。以下按“现在申请”和“后续再申请”分层。
> 默认地域：华东1（杭州）。

## 1. 现在就要申请（当前正式路径）

| 资源 | 业务作用（通俗版） | MVP 必要性 | 预期效果 | 预估成本（月） |
|---|---|---|---|---|
| VPC + 交换机 + 安全组 | 给你的服务划一个私有网络和防火墙规则 | 建议必备 | 服务可互通且更安全 | 通常免费 |
| FC Web 函数 | 承载正式后端 API | 必备 | iOS 能通过正式入口访问 API | 按实际消耗收费 |
| RDS PostgreSQL Serverless | 正式数据库，自动启停 | 必备 | 数据可持久化、可备份、低负载时自动休眠 | 按实际消耗收费 |
| OSS | 存头像/封面等图片 | 必备 | 图片统一存储、可扩展 CDN | 按量 |
| CloudMonitor | 服务/数据库告警 | 建议必备 | 出故障时能第一时间通知 | 低至免费 |

## 2. 后续再申请（等正式域名接入时）

| 资源 | 业务作用 | MVP 必要性 |
|---|---|---|
| 域名 + ICP 备案 | 国内用户稳定访问与合规 | 正式域名接入时必备 |
| HTTPS 证书 | iOS 强制 HTTPS | 正式域名接入时必备 |
| SLS 日志服务 | 集中日志检索与审计 | 强烈建议 |

## 3. 预算建议（华东1，预发）

- 低配联调区间：`¥100-400 / 月`
- 主要构成：FC + RDS Serverless + OSS + 基础监控

## 4. 申请前准备清单

1. 阿里云实名认证（个人或企业）。
2. 预算上限（建议先设 `¥500/月` 告警阈值）。
3. 运营联系人手机号/邮箱（接告警）。
4. RAM 子账号（不要长期使用主账号）。
5. 如准备正式域名接入：域名主体与备案主体一致。

## 5. 操作顺序（推荐）

1. 先建 VPC/安全组。
2. 再建 RDS PostgreSQL Serverless。
3. 再建 OSS。
4. 再部署 FC Web 函数。
5. 最后处理域名备案、HTTPS 与正式自定义域名接入。
