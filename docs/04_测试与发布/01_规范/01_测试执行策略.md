# 测试执行策略

[PROTOCOL]: 变更时更新此头部，然后检查 agents.md

**版本**: v3.0
**日期**: 2026-03-22

## 1. 目标
1. 快速反馈：优先在 3 至 5 分钟内判断“本地后端 + 模拟器”是否明显损坏。
2. 风险匹配：后端规则、iOS 功能、UI 行为按 lane 分层执行，避免无效全量。
3. 可重复联调：核心业务流优先在“本机后端 + 模拟器”环境稳定复现；正式入口仅保留 FC 默认域名与 `TestFlight` 复用链路。

## 2. 分层策略
### 2.1 `smoke-local`（默认快检 / PR 主门禁）
1. iOS：治理检查 + `BianLunMiaoTests` 全量 + 最短 UI 冒烟。
2. UI 目标：只回答“App 有没有明显坏”。
3. UI 范围：启动 Mock 首页、登录门禁、5 Tab 可达。

### 2.2 `local-remote`（本机联调）
1. 目标：回答“本机后端联调是否正常”。
2. UI 范围：远端调试会话建立、主 Tab 可达、1 条最小写操作闭环。
3. 运行前提：本机后端已启动且允许 debug-token。

### 2.3 `prod-data`（正式服默认域名数据专项）
1. 目标：回答“正式后端默认域名下的核心写数据链路与日历同步是否可用”。
2. UI 范围：队伍搜索与入队申请、队伍 CRUD、赛事 CRUD、场次 CRUD、日历同步。
3. 约束：只用于正式服预演与真机验收，不纳入默认门禁。
4. 命令：`BLM_TEST_REMOTE_BASE_URL=<fc-default-domain> bash scripts/ios_ui_lane.sh prod-data`

### 2.4 `device-special`（真机专项）
1. 目标：验证真机安装、ATS、Apple 登录入口与系统授权链路。
2. 范围：只保留真机特有链路，不纳入默认 PR/快检。

### 2.5 `specialized`（素材 / 启动性能专项）
1. 目标：沉淀 App Store 截图、启动性能与 Launch 专项产物。
2. 范围：只保留素材和性能采样，不纳入默认 PR/快检主门禁。
3. 命令：`bash scripts/ios_ui_lane.sh specialized`

### 2.6 `full-local`（主干健康 / 发布前本地全检）
1. 后端基线：`make lint` + 全量 `pytest`。
2. iOS：治理检查 + 单测 + `smoke-local` + 本地 Mock 长链路功能回归。
3. 日程页回归需覆盖单日详情暗色可达与 0-24 全天时间轴断言。
4. 专项：截图产物、启动性能、真机 Apple 登录独立执行，不阻塞功能门禁。

## 3. iOS 联调自动化约束
### 3.1 Apple 登录
1. Apple 登录不纳入模拟器自动化。
2. Apple 登录只保留真机人工验收，用于最终线上确认。

### 3.2 lane 环境变量
1. `BLM_API_BASE_URL` 仅作为显式覆盖，不再承载“默认是本地还是正式”的隐式策略；值应为根地址，不再拼接 `/api/v1`。
2. `BLM_TEST_REMOTE_BASE_URL` 仅供 UI lane 脚本向 `local-remote` / `prod-data` 注入目标地址，既可以是本机，也可以是 FC 默认域名。
3. `BLM_DEBUG_PUBLIC_ID` 仅供远端数据专项为每个用例注入独立测试用户，避免共享数据相互污染。
4. `BLM_UI_TEST_TARGET_TEAM_PUBLIC_ID` 与 `BLM_UI_TEST_TARGET_TEAM_NAME` 仅供数据专项注入固定入队目标。
5. `BLM_UI_TEST_ALLOW_REMOTE`、`BLM_ENABLE_DEBUG_SESSION_FALLBACK` 仅在 `local-remote` / `prod-data` 开启。
6. `BLM_FORCE_NEW_USER_FLOW` 默认关闭，仅在显式专项调试时开启。
7. `BLM_UI_TEST_COLOR_SCHEME` 仅供 UI 自动化覆盖测试色彩方案，当前至少支持 `dark`。

### 3.3 模拟器本机联调
1. `local-remote` 固定环境变量为：
   - `BLM_UI_TEST_MODE=1`
   - `BLM_UI_TEST_RESET_STATE=1`
   - `BLM_USE_MOCK_DATA=0`
   - `BLM_UI_TEST_ALLOW_REMOTE=1`
   - `BLM_ENABLE_DEBUG_SESSION_FALLBACK=1`
   - `BLM_API_BASE_URL=http://127.0.0.1:8788`
2. 仅在测试环境允许的附加开关：
   - `BLM_UI_TEST_REMOTE_TEAM_CREATE=1`
   - `BLM_UI_TEST_TEAM_NAME`
   - `BLM_UI_TEST_TEAM_SLOGAN`

### 3.4 本机后端
1. 自动化联调统一使用独立 SQLite。
2. 本机服务需要显式开启 `ENABLE_DEBUG_TOKEN=true`。
3. 正式联调前必须先完成数据库 schema 初始化，再启动后端服务。

## 4. UI 成功态锚点
1. 主 Tab：`main_tab_team|tournament|schedule|message|my`
2. Tab 根：`team_list_root`、`tournament_list_root`、`schedule_root`、`message_hub_root`、`my_hub_root`
3. 队伍详情根：`team_detail_root`
4. 队伍创建成功 toast：`team_create_success_toast`
5. 赛事详情根：`tournament_detail_root`
6. 场次详情根：`match_detail_root`
7. 消息详情根：`message_detail_root`

## 5. 门禁规则
1. 快检失败禁止合并。
2. 全检失败禁止发布。
3. Apple 登录人工项未完成时，不得标记“线上验收完成”。
4. 放宽规则或新增测试专用开关时，必须更新本文件变更日志。
5. `local-remote` 仅允许本机后端联调，不得指向云端测试服。
6. `prod-data` 允许写正式后端默认域名预演数据，但只用于正式迁移演练，执行前必须确认不迁旧正式数据。
