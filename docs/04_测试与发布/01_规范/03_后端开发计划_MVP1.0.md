# 后端开发计划（MVP 1.0）

[PROTOCOL]: 变更时更新此头部，然后检查 agents.md

**版本**: v1.0
**日期**: 2026-02-17


## 1. 目标

- 将 iOS 本地 `AppStore + MockData` 业务迁移到独立后端。
- 保持现有前端交互不重做，优先替换数据读写来源。
- 支持 iOS 与未来 Android 复用同一套 API 契约。

## 2. 里程碑与 DoD

### Phase 0：工程初始化（已完成）
- 完成后端目录、FastAPI 工程骨架。
- 完成数据库模型与初始迁移。
- 完成核心文档与运行指南。

DoD：`docker compose up` + `alembic upgrade head` + `uvicorn` 可启动。

### Phase 1：Auth + User + Team/JoinRequest（已完成 MVP 实现）
- Apple 登录（联调模式）+ Debug Token。
- 用户信息查询/修改/搜索。
- 队伍创建、编辑、我的队伍、发现队伍。
- 入队申请提交、审批通过/拒绝。

DoD：可跑通“登录 -> 创建队伍 -> 申请入队 -> 审批”。

### Phase 2：Tournament + Match + Roster + Result（已完成 MVP 实现）
- 赛事创建/列表/详情/编辑。
- 场次创建/编辑/队伍指派。
- 阵容指派与合法性校验。
- 状态流转与赛果录入。

DoD：可跑通“创建赛事 -> 新建场次 -> 指派阵容 -> 结束并录结果”。

### Phase 3：Schedule + Message + Media + 预发联调（已完成基础实现）
- 日程聚合查询与来源管理。
- 消息流拉取与 ack。
- 头像/封面上传 token。
- 预发部署清单文档化。

DoD：iOS 可读取日程、消息、上传 token。

### Phase 4：稳定性收口（待执行）
- 补充 CI、回归测试、监控告警。
- 增加审计日志与发布门禁。
- 契约冻结并版本化发布。

## 3. 风险与依赖

1. Apple token 真实校验尚未接入，当前为联调模式。
2. OSS 预签名当前为 mock 形式，需在预发接入真实 RAM/STS。
3. 当前未启用 Redis 业务逻辑，仅完成基础拓扑预留。
4. 需 iOS 侧新增环境切换并注入真实 token。

## 4. 交付清单

- 可运行服务
- Alembic 迁移
- OpenAPI 文档
- 后端与云资源操作指引
