# 发布与回滚规范

[PROTOCOL]: 变更时更新此头部，然后检查 agents.md

**版本**: v1.1
**日期**: 2026-02-23

## 1. 发布流程
1. 预发环境通过最小联调后，方可进入生产发布。
2. 发布批次必须记录版本、时间、负责人、回滚点。
3. 配置变更与代码发布必须可追踪。

## 2. 上线前检查
1. API 契约与客户端版本匹配。
2. 数据库迁移可回滚。
3. 监控与告警已配置。
4. 关键路径冒烟通过。

## 3. 回滚策略
1. 应用层：支持回退到上一个稳定镜像版本。
2. 数据层：每个迁移必须提供 `downgrade` 或明确不可逆说明。
3. 契约层：若变更破坏兼容，需提供降级开关或灰度策略。

## 4. 事故处理
1. 出现 P0/P1 事故时立即停止扩散发布。
2. 先恢复服务，再补完整根因分析与预防措施。
3. 事故复盘必须回写到文档。

## 5. Android 测试包发布规则（staging）
1. 构建产物必须包含：`app-debug.apk` 与 `SHA256` 校验值。
2. 测试包分发统一上传到阿里云 OSS `stg/apk/android/` 路径，并记录可下载链接。
3. 构建前必须校验 `BuildConfig.API_BASE_URL` 指向测试环境，禁止 `localhost/127.0.0.1`。
4. 回滚必须保留上一版 APK 链接与 `SHA256`，出现安装失败或关键链路故障时 15 分钟内切回上一版。
5. 发布记录必须写明：版本号、构建时间、上传路径、下载链接、校验值、回滚点。
