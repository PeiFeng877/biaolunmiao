# 测试执行策略

[PROTOCOL]: 变更时更新此头部，然后检查 agents.md

**版本**: v1.1
**日期**: 2026-03-03

## 1. 目标
1. 快速反馈：日常改动优先暴露真实阻塞，而不是把问题留到发布前。
2. 风险匹配：后端规则、iOS 功能、UI 行为分层执行，避免无效全量。
3. 可重复联调：除 Apple 登录外，核心业务流应能在“本机后端 + 模拟器”环境稳定复现。

## 2. 分层策略
### 2.1 快检（PR 主门禁）
1. iOS：治理检查 + 单测 + 高风险 UI 冒烟。
2. 后端：`make lint` + 核心 `pytest` 用例。
3. 文档：契约与测试策略文档同构更新。

### 2.2 全检（主干与发布门禁）
1. 后端基线：`make lint` + 全量 `pytest`。
2. iOS 单测：`BianLunMiaoTests` 全量通过。
3. iOS UI：按功能域拆分执行，至少覆盖队伍、赛事/场次、日程、消息、我的/资料。
4. 联调：除 Apple 登录外，优先用本机后端承接模拟器自动化。

## 3. iOS 联调自动化约束
### 3.1 Apple 登录
1. Apple 登录不纳入模拟器自动化。
2. Apple 登录只保留真机人工验收，用于最终线上确认。

### 3.2 模拟器远端联调
1. UI 测试默认允许通过本机后端跳过 Apple 登录前置。
2. 测试环境变量固定为：
   - `BLM_UI_TEST_MODE=1`
   - `BLM_UI_TEST_RESET_STATE=1`
   - `BLM_USE_MOCK_DATA=0`
   - `BLM_UI_TEST_ALLOW_REMOTE=1`
   - `BLM_ENABLE_DEBUG_SESSION_FALLBACK=1`
   - `BLM_API_BASE_URL=http://127.0.0.1:8000/api/v1`
3. 仅在测试环境允许的附加开关：
   - `BLM_UI_TEST_REMOTE_TEAM_CREATE=1`
   - `BLM_UI_TEST_TEAM_NAME`
   - `BLM_UI_TEST_TEAM_SLOGAN`

### 3.3 本机后端
1. 自动化联调统一使用独立 SQLite。
2. 本机服务需要显式开启 `ENABLE_DEBUG_TOKEN=true`。
3. 远端联调前必须先迁移数据库，再启动 `uvicorn`。

## 4. UI 成功态锚点
1. 队伍详情根：`team_detail_root`
2. 队伍创建成功 toast：`team_create_success_toast`
3. 赛事详情根：`tournament_detail_root`
4. 场次详情根：`match_detail_root`
5. 消息详情根：`message_detail_root`
6. 资料页与日程页继续沿用既有稳定 accessibility id。

## 5. 门禁规则
1. 快检失败禁止合并。
2. 全检失败禁止发布。
3. Apple 登录人工项未完成时，不得标记“线上验收完成”。
4. 放宽规则或新增测试专用开关时，必须更新本文件变更日志。
