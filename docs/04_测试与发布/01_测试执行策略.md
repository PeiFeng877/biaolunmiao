# 测试执行策略

[PROTOCOL]: 变更时更新此头部，然后检查 agents.md

**版本**: v1.0
**日期**: 2026-02-17

## 1. 目标
1. 快速反馈：日常提交应在短时间得到可行动反馈。
2. 风险匹配：小改动跑快检，大改动跑全检。
3. 稳定交付：发布前覆盖关键跨端链路。

## 2. 分层策略
### 2.1 快检（PR 主门禁）
1. iOS：治理检查 + lint + 单测 + 风险触发 UI 冒烟。
2. 后端：`ruff check` + 核心接口测试。
3. 文档：契约文档变更完整性检查。

### 2.2 全检（主干与发布门禁）
1. iOS：全量测试。
2. 后端：全量 `pytest` + 关键联调用例。
3. 联调：至少验证认证、队伍申请、赛事排阵、日程查询、消息确认。

## 3. 约束
1. 快检失败禁止合并。
2. 全检失败禁止发布。
3. 放宽规则必须更新本文件变更日志。
