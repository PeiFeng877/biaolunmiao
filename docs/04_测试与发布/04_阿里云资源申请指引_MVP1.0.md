# 阿里云资源申请指引（MVP 1.0，小白版）

[PROTOCOL]: 变更时更新此头部，然后检查 agents.md

**版本**: v1.0
**日期**: 2026-02-17


> 目标人群：非技术背景。以下按“现在申请”和“上线前申请”分层。
> 默认地域：华东1（杭州）。

## 1. 现在就要申请（联调必须）

| 资源 | 业务作用（通俗版） | MVP 必要性 | 预期效果 | 预估成本（月） |
|---|---|---|---|---|
| VPC + 交换机 + 安全组 | 给你的服务划一个私有网络和防火墙规则 | 必备 | 服务可互通且更安全 | 通常免费 |
| ECS | 放后端程序的云服务器 | 必备 | API 能被 iOS 访问 | ¥120-220 |
| RDS PostgreSQL | 正式数据库，替代本地 Mock | 必备 | 数据可持久化、可备份 | ¥120-260 |
| Redis(Tair) | 缓存和会话加速 | 建议必备 | 提升响应速度，后续扩展方便 | ¥40-120 |
| OSS | 存头像/封面等图片 | 必备 | 图片统一存储、可扩展 CDN | ¥5-40 |
| ACR | 存放 Docker 镜像 | 建议必备 | 发布更标准、可回滚 | ¥0-30 |
| CloudMonitor | 服务器/数据库告警 | 必备 | 出故障时能第一时间通知 | ¥0-30 |

## 2. 上线前申请（正式商用）

| 资源 | 业务作用 | MVP 必要性 |
|---|---|---|
| 域名 + ICP 备案 | 国内用户稳定访问与合规 | 正式上线必备 |
| HTTPS 证书 | iOS 强制 HTTPS | 正式上线必备 |
| SLS 日志服务 | 集中日志检索与审计 | 强烈建议 |
| ALB（可选） | 高并发和流量治理 | 可选 |

## 3. 预算建议（华东1，预发）

- 低配联调区间：`¥300-800 / 月`
- 主要构成：ECS + RDS + Redis + OSS + 基础监控

## 4. 申请前准备清单

1. 阿里云实名认证（个人或企业）。
2. 预算上限（建议先设 `¥800/月` 告警阈值）。
3. 运营联系人手机号/邮箱（接告警）。
4. RAM 子账号（不要长期使用主账号）。
5. 如准备正式上线：域名主体与备案主体一致。

## 5. 操作顺序（推荐）

1. 先建 VPC/安全组。
2. 再建 RDS/Redis/OSS。
3. 再建 ECS + 部署后端。
4. 配置监控告警。
5. 最后处理域名备案与 HTTPS。
