# 赛事 MVP 交互 PRD (Tournament Interaction MVP)

**版本**: v1.0
**日期**: 2026-02-03
**核心目标**: 定义赛事创建、赛程编排、人员指派及日程生成的最小闭环流程。

---

## 1. 创建赛事 (Create Tournament)

### 1.1 入口
*   首页“赛事” Tab -> 右上角 “+” 号。

### 1.2 创建表单 (Form)
*   **名称**: 必填。
*   **图片/头像**: 选填（提供默认图）。
*   **介绍**: 选填。

### 1.3 创建后状态
*   创建成功后，直接进入“赛事管理后台”视图。
*   当前用户默认为 **创建人 (Creator)**。

### 1.4 管理员设置 (Admin Setting)
*   **入口**: 赛事详情页 -> “设置” -> “管理员管理”。
*   **逻辑**: 创建人可以通过搜索 User ID 添加管理员。
*   **权限**: 管理员拥有除“设置管理员”以外的所有权限（如审核报名、排赛程、录入结果）。

---

## 2. 赛程编排 (Schedule Management)

### 2.1 赛程层级
赛事的结构为两层：
1.  **赛事 (Tournament)**: 大容器 (如“2026星火杯”)。
2.  **赛程/场次 (Match)**: 具体比赛单元 (如“初赛第一场”)。

### 2.2 创建赛程 (Create Match)
*   **入口**: 赛事管理页 -> “赛程表” -> “新建赛程”。
*   **步骤**:
    1.  **赛程基础**:
        *   **名称**: 如“初赛第一场”。
        *   **ID**: 系统自动生成，用于内部关联。
    2.  **赛制设定**:
        *   **模式**: 下拉选择 (1v1, 2v2, 3v3, 4v4)。这决定了后续队伍需要指派几个人。
    3.  **时间设定**:
        *   选择具体的开始时间 (Date & Time)。
    4.  **对阵双方 (Matchup)**:
        *   **前提**: 必须有已通过审核的报名队伍。
        *   **操作**: 从“已报名队伍池”中选择 Team A 和 Team B。
    5.  **工作人员 (Staff)**:
        *   **主席**: 必选 1 人（搜索用户添加）。
        *   **评委**: 可选 N 人（搜索用户添加）。

### 2.3 状态流转
*   赛程创建完成后，默认状态为 **“未开始” (Scheduled)**。
*   **触发动作**: 系统自动向 Team A 和 Team B 的队长/管理员发送“待指派”通知。

---

## 3. 队员指派 (Rostering) - *核心闭环*

### 3.1 接收任务
*   Team A 的队长收到通知，进入该场次详情页。
*   看到状态提示：“请指派上场队员 (需 4 人)”。

### 3.2 指派操作
*   **界面**: 展示当前队伍的所有成员列表。
*   **交互**: 勾选 4 名成员，并标记辩位（一辩、二辩、三辩、四辩）。
*   **提交**: 点击确认。

### 3.3 数据闭环 (日程生成)
*   提交成功瞬间，系统执行以下操作：
    1.  **队员视角**: 被选中的 4 名队员，其 App 首页“我的日程”出现该比赛，且同步写入系统日历。
    2.  **赛事视角**: 该场次状态变为“名单已确认” (Ready)。

---

## 4. 赛事执行与结算 (Execution & Settlement)

### 4.1 状态管理
*   管理员手动控制场次状态：
    *   **未开始** -> **进行中** (比赛开始)。
    *   **进行中** -> **已结束** (比赛完结)。

### 4.2 结果录入 (Result Entry)
*   仅在“已结束”状态下可操作。
*   **胜负**: 选择胜方 (Team A 或 Team B)。
*   **评分**:
    *   录入每位评委对 A 队和 B 队的总分。
    *   (MVP 阶段暂不涉及复杂的单项分，仅录总分)。

---

## 5. 页面流转图 (User Flow)

1.  **[赛事方]** 创建赛事 -> 开启报名 -> 等待队伍报名 -> 审核通过。
2.  **[赛事方]** 新建赛程 -> 选定 A/B 队 + 时间 + 赛制 -> 发布。
3.  **[队长]** 收到通知 -> 进入赛程 -> 勾选上场队员 -> 提交。
4.  **[队员]** 收到日程推送 -> 系统日历自动创建日程。
5.  **[赛事方]** 比赛结束 -> 修改状态为“已结束” -> 录入胜负/分数。
