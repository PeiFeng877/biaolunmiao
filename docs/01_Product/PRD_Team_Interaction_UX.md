# 队伍交互与体验 PRD (Team Interaction & UX)

**版本**: v1.0
**日期**: 2026-02-03
**关联文档**: [PRD_Core_Logic_Rules.md](./PRD_Core_Logic_Rules.md)
**核心目标**: 定义队伍模块的创建、管理、成员交互及权限逻辑，规范 UI 页面流转。

---

## 1. 实体数据定义 (Entity Schema)

### 1.1 队伍 (Team)
*   **UUID (Internal)**: 系统后台生成的 32 位唯一标识符，用于数据库关联，**用户不可见**。
*   **Public ID (Display)**: 系统自动分配的短 ID（如 6-8 位数字），**唯一，用于搜索和展示**。
*   **名称 (Name)**: 允许重复（如可能有 10 个“复仇者队”），依靠 Public ID 区分。
*   **简介 (Bio)**: 文本描述。
*   **头像 (Avatar)**: 图片 URL。

### 1.2 用户 (User)
*   **UUID (Internal)**: 系统后台唯一标识。
*   **User ID (Custom)**: **用户自定义的唯一标识**（字母+数字组合），用于搜索和精准定位。
*   **昵称 (Nickname)**: 允许重复。

## 2. 交互流程：创建队伍 (Creation Flow)

### 2.1 入口 (Entry Point)
*   **位置**: App 底部 Tab栏 -> **“我的 (Profile)”** -> **“我的队伍”** 卡片区域。
*   **交互**: 点击卡片右上角的 **“+”** 号按钮。

### 2.2 创建界面 (Creation UI)
*   **形式**: **模态弹窗 (Modal Sheet)**。保证用户不脱离当前上下文。
*   **表单字段**:
    1.  **头像上传** (点击占位图选择)。
    2.  **队伍名称** (必填)。
    3.  **队伍 ID** (系统自动生成预填，不可修改，展示给用户看，告知这是以后的搜索码)。
    4.  **简介** (选填)。
*   **操作**:
    *   点击“创建”按钮。
    *   **反馈**: Loading 转圈 -> 成功 Toast 提示 -> 模态窗自动关闭。

### 2.3 后置流程 (Post-Action)
*   窗口关闭后，自动跳转至新创建队伍的 **“队伍详情/管理页”**。

## 3. 交互流程：成员管理 (Member Interaction)

### 3.1 邀请加入 (Invite Flow) - *主动方：队长/管理*
*   **入口**: 队伍详情页 -> 成员列表 -> “邀请成员”按钮。
*   **搜索**:
    *   输入框：支持输入用户的 **昵称** 或 **User ID**。
    *   结果列表：展示头像、昵称、User ID。
*   **发送邀请**:
    *   点击“邀请”。
    *   **状态变更**: 该用户在列表中显示为“邀请中 (Pending)”。
*   **防骚扰逻辑 (Anti-Spam)**:
    *   如果用户拒绝邀请，状态变为“已拒绝”。
    *   **规则**: 若用户拒绝邀请，队长在 **24小时内** 不可再次向该用户发送邀请（按钮置灰）。

### 3.2 申请加入 (Apply Flow) - *主动方：路人*
*   **入口**: 
    *   方法 A: 首页/广场 -> 搜索栏 -> 搜队伍名或 Public ID。
    *   方法 B: 赛事详情页 -> 参赛队伍列表 -> 点击某队伍头像。
*   **操作**:
    *   进入“队伍公开主页”。
    *   点击底部 **“申请加入”** 按钮。
*   **反馈**: 按钮变为“申请已发送”，等待审批。

### 3.3 审批流程 (Approval)
*   **通知**: 队长/管理员会在“消息/待办”中心收到红点通知。
*   **操作**:
    *   “同意” -> 用户正式入队，系统日历同步权限校验。
    *   “拒绝” -> 申请单归档。

## 4. 权限管理逻辑 (Permission Matrix)

| 权限操作 | 队长 (Owner) | 管理员 (Admin) | 队员 (Member) |
| :--- | :---: | :---: | :---: |
| **创建队伍** | ✅ | ✅ | ✅ |
| **解散队伍** | ✅ | ❌ | ❌ |
| **转让队长** | ✅ | ❌ | ❌ |
| **指派/撤销管理员** | ✅ | ❌ | ❌ |
| **邀请成员** | ✅ | ✅ | ❌ |
| **审批入队申请** | ✅ | ✅ | ❌ |
| **移除成员 (踢人)** | ✅ (含管理) | ✅ (仅队员) | ❌ |
| **报名赛事** | ✅ | ✅ | ❌ |
| **编辑队伍资料** | ✅ | ✅ | ❌ |

*注：一个人可以是 A 队的队长，同时是 B 队的队员。权限是绑定在“人-队”关系上的，而非全局属性。*

## 5. 页面结构概览 (UI Map)

1.  **我的队伍列表页** (Profile -> My Teams)
    *   List 视图：展示我加入的所有队伍。
    *   区分标记：“我创建的”、“我管理的”、“我加入的”。
2.  **队伍详情页 (管理视角)** (Team Detail - Admin View)
    *   **Header**: 头像、名称、ID、简介（支持编辑）。
    *   **Dashboard**: 待处理申请 (Badge 提示)。
    *   **Action Area**: 报名中的比赛、即将开始的赛程。
    *   **Member List**: 成员列表（按角色分组：队长 > 管理 > 队员）。点击成员头像弹出操作菜单（设为管理/踢出）。
    *   **Footer**: 退出队伍/解散队伍（红字）。
3.  **队伍公开页 (路人视角)** (Team Public Profile)
    *   展示基础信息。
    *   展示历史战绩/荣誉墙。
    *   底部大按钮：“申请加入”。
