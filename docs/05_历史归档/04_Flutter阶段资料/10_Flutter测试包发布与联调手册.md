# Flutter 测试包发布与联调手册（staging）

[PROTOCOL]: 变更时更新此头部，然后检查 agents.md

**版本**: v1.0
**日期**: 2026-03-05
**状态**: 历史归档（2026-03-14 起退出现行 SSOT）

## 1. 目标
1. 产出可安装的 Flutter Android 测试包。
2. 全量请求连接阿里云测试环境，不连接本机。
3. 验证 Flutter 与 iOS 对同账号数据的跨端互通。

## 2. 前置条件
1. 后端环境为 `staging`，且测试登录链路已开启。
2. API 基址可访问：`http://120.55.115.147/api/v1`（或最新 stg 域名）。
3. Flutter 构建配置已指向测试环境并开启本地地址拦截。

## 3. 发布步骤
1. 构建 debug APK：`flutter build apk --debug`
2. 计算校验值：`shasum -a 256 build/app/outputs/flutter-apk/app-debug.apk`
3. 上传到 OSS：建议路径 `stg/apk/flutter/<version>/app-debug.apk`
4. 记录发布信息：版本、构建时间、下载链接、SHA256、回滚点。

## 4. 联调验收清单
1. 登录：测试账号可成功进入主页面。
2. 队伍：创建队伍、提交入队申请、审批链路可用。
3. 赛事：创建赛事、创建场次、录入结果可用。
4. 日程：来源管理与系统日历同步可用。
5. 消息：消息拉取与确认接口可用。
6. 我的：昵称/头像更新可回写后端。
7. 跨端互通：Flutter 与 iOS 对同一账号操作可互见。

## 5. 回滚策略
1. 保留上一版 APK 下载链接与 SHA256。
2. 出现 P1 问题时立即下线当前链接并切回上一版。
3. 回滚后补充问题原因、影响范围、修复计划。

## 6. 风险提示
1. 测试登录链路仅限测试环境，禁止带入生产。
2. 若测试域名受访问策略影响，可临时使用 stg 公网 IP。
3. 发布包必须明确标注“测试版”，避免误装为正式版本。
