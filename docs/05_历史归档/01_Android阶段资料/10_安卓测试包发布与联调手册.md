# 安卓测试包发布与联调手册（staging）

[PROTOCOL]: 变更时更新此头部，然后检查 agents.md

**版本**: v1.3
**日期**: 2026-02-23
**状态**: 历史归档（Android 独立仓阶段）

## 1. 目标
1. 产出可安装的 Android 测试 APK。
2. 全量请求连接阿里云测试环境，不连接本机。
3. 通过测试手机号登录验证跨端数据互通。

## 2. 前置条件
1. 后端环境为 `staging`，且 `ENABLE_TEST_PHONE_LOGIN=true`。
2. API 基址可访问：`http://120.55.115.147/api/v1`。
3. Android 构建配置已写入测试基址并启用本地地址拦截。

## 3. 发布步骤
1. 构建 `debug` 包：`./gradlew :app:assembleDebug`。
2. 计算校验值：`shasum -a 256 app/build/outputs/apk/debug/app-debug.apk`。
3. 上传到 OSS：建议路径 `stg/apk/android/<version>/app-debug.apk`。
4. 记录发布信息：版本、构建时间、下载链接、SHA256、回滚点。

## 4. 联调验收清单
1. 登录：手机号 11 位 + 任意非空验证码可登录成功。
2. 队伍：创建队伍、提交入队申请、审批链路可用。
3. 赛事：创建赛事、创建场次、录入结果可用。
4. 日程：来源管理与同步到系统日历可用。
5. 消息：消息列表可拉取，确认接口可用。
6. 我的：昵称/头像更新可回写后端。
7. 跨端互通：Android 与 iOS 对同一账号的操作可互见。

## 5. 回滚策略
1. 保留上一版 APK 下载链接与 SHA256。
2. 出现 P1 问题时立即下线当前链接，切换回上一版。
3. 回滚后补充问题原因、影响范围、修复计划。

## 6. 风险提示
1. 该登录链路仅用于测试环境，不可用于生产环境。
2. 若测试域名受访问策略影响，可使用 staging 公网 IP 直连。
3. 发布包必须明确标注“测试版”，避免外部误装为正式版本。
