# 测试执行策略

[PROTOCOL]: 变更时更新此头部，然后检查 agents.md

**版本**: v2.0
**日期**: 2026-03-19
**定位**: BianLunMiao iOS UI 自动化与治理门禁执行规范。

## 1. 目标定义
1. 质量底线：关键流程不可被小改动破坏。
2. 反馈速度：默认优先回答“本地模拟器或本机后端是否坏了”。
3. 风险匹配：把 UI 自动化拆成 lane，避免截图、真机专项和本地快检互相干扰。

## 2. 执行分层
### 2.1 `smoke-local`
1. 触发：PR 快检、日常本地自检。
2. 内容：治理脚本、SwiftFormat/SwiftLint、`BianLunMiaoTests`、最短 UI smoke。
3. 命令：`bash scripts/ios_ui_lane.sh smoke-local`
4. 目标：3 至 5 分钟内确认 App 是否明显损坏。

### 2.2 `full-local`
1. 触发：主干健康、发布前本地功能回归。
2. 内容：`smoke-local` + 本地 Mock 长链路 UI 功能测试。
3. 命令：`bash scripts/ios_ui_lane.sh full-local`

### 2.3 `local-remote`
1. 触发：本机后端联调。
2. 内容：调试会话建链、主 Tab 可达、1 条最小写操作闭环。
3. 命令：`BLM_TEST_REMOTE_BASE_URL=http://127.0.0.1:8000/api/v1 bash scripts/ios_ui_lane.sh local-remote`

### 2.4 `stg-smoke`
1. 触发：预发环境健康检查。
2. 内容：只读或低副作用远端 UI 验证。
3. 命令：`BLM_TEST_REMOTE_BASE_URL=http://120.55.115.147/api/v1 bash scripts/ios_ui_lane.sh stg-smoke`

### 2.5 `device-special`
1. 触发：真机专项、Apple 登录人工前置验证。
2. 内容：安装启动、登录入口、ATS/系统授权链路。
3. 命令：`IOS_UI_DESTINATION='id=<device-udid>' bash scripts/ios_ui_lane.sh device-special`

### 2.6 `specialized`
1. 触发：截图产物、启动性能、Launch 专项。
2. 内容：`BianLunMiaoSpecializedUITests` 与 `BianLunMiaoUITestsLaunchTests`
3. 命令：`bash scripts/ios_ui_lane.sh specialized`

## 3. 规范要求
1. 新增 UI 测试必须先归属 lane，再决定是否进入默认门禁。
2. 默认门禁只包含 `smoke-local`；`local-remote`、`stg-smoke`、`device-special` 必须显式触发。
3. `BLM_API_BASE_URL` 只作为显式覆盖，不再被 UI 测试默认写死。
4. 远端测试默认关闭“强制新用户资料流”，避免落点抖动。
5. 主 Tab 与关键页面必须提供稳定 accessibility id，不再依赖中文文案定位。
6. 治理脚本始终属于 Hard Gate，不能降级为告警。

## 4. 后续优化路线
1. 在 `smoke-local` 上进一步压缩时长，目标稳定在 3 分钟以内。
2. 按 lane 追加 `.xctestplan`，把脚本分片策略收口到 Xcode 配置层。
3. 持续验证 `Maestro` PoC 的收益，若分片提速不明显则继续以 `XCTest/XCUITest` 为唯一主线。
